Solidity的数值陷阱1. 什么是整数溢出？计算机中的整数有固定大小。以太坊使用256位整数（uint256），这意味着它的取值范围是0到2^256-1。超过这个范围，就会发生溢出——数值突然变成一个很小的数，或者归零。 打个比方：汽车的里程表跑到999999公里后，再加1公里就会归零显示0公里。整数溢出就是这种”归零”现象。 整数溢出分两种： • 上溢（Overflow）：最大值+1=0或最小值• 下溢（Underflow）：最小值-1=最大值 Solidity 0.8之前：不报错，静默归零Solidity 0.8之后：自动revert 2. 溢出攻击示例用一个简单的代币合约来演示溢出攻击。 OverflowToken.sol 12345678910111213// SPDX-License-Identifier: MITpragma solidity ^0.7.6; // 旧版本，有溢出漏洞contract OverflowToken { mapping(address => uint256) balances; func...
