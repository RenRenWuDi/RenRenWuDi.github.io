区块链是如何工作的？
哈希、Merkle树与链式结构
为什么篡改历史几乎不可能
💡 你将学到：哈希函数、Merkle树、区块结构、链式验证原理
⏱ 阅读时间：约12分钟
1. 哈希：区块链的"数字指纹"
上一篇讲了比特币的UTXO模型。但你有没有想过：这些交易数据是怎么被安全地存储和验证的？
答案是：哈希函数。
哈希函数是什么？

把任意长度的数据，变成固定长度的"指纹"。

特点：
• 输入改变1个字节，输出完全不同（雪崩效应）
• 无法从哈希值反推原始数据（单向性）
• 不同数据几乎不可能产生相同哈希（抗碰撞）
代码示例：SHA256哈希
merkle-tree.js
const crypto = require('crypto')

// SHA256 哈希函数
function sha256(data) {
return crypto.createHash('sha256').update(data).digest('hex')
}

// 测试：同样的输入，同样的输出
console.log(sha256('Hello'))
// 输出: 185f8db32271fe25f561a6fc938b2e264306ec304eda518007d1764826381969

// 改变一个字符，输出完全不同
console.log(sha256('hello'))
// 输出: 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

哈希在区块链中的作用
交易哈希：每笔交易都有唯一"身份证号"
区块哈希：整个区块内容的指纹
链式连接：当前区块包含前一区块的哈希
2. Merkle树：高效验证大量数据
一个区块里有几千笔交易。如何快速验证某笔交易是否在这个区块里？
如果遍历所有交易，太慢了。Merkle树解决了这个问题。
Merkle树是什么？

一种二叉树结构，叶子节点是交易哈希，父节点是子节点哈希的哈希。

好处：只需少量哈希值（Merkle路径），就能验证某笔交易是否存在。
Merkle树结构示意
merkle-tree.js
[Root Hash]           <- 区块头只存这个
/         \
[Hash AB]     [Hash CD]
/     \       /     \
[A]     [B]   [C]     [D]   <- 交易哈希（叶子）

// 验证交易C是否存在：
// 只需要知道：Hash D + Hash AB
// 计算：Hash(C+D) → 与 Hash AB 组合 → 对比 Root

代码示例：构建Merkle树
merkle-tree.js
const crypto = require('crypto')

function sha256(data) {
return crypto.createHash('sha256').update(data).digest('hex')
}

// 构建Merkle树
function buildMerkleTree(leaves) {
if (leaves.length === 1) return leaves[0]

const nextLevel = []
for (let i = 0; i < leaves.length; i += 2) {
const left = leaves[i]
const right = leaves[i + 1] || leaves[i] // 奇数复制最后一个
nextLevel.push(sha256(left + right))
}

return buildMerkleTree(nextLevel)
}

// 4笔交易
const txs = ['tx1', 'tx2', 'tx3', 'tx4']
const leaves = txs.map(sha256)
console.log('Merkle Root:', buildMerkleTree(leaves))

3. 区块结构：头部 + 交易体
区块链的"区块"长什么样？
区块 = 区块头 + 交易列表

区块头（80字节）：
• 前一区块哈希（32字节）
• Merkle根（32字节）
• 时间戳（4字节）
• 难度目标（4字节）
• 随机数Nonce（4字节）

交易列表：该区块包含的所有交易
代码示例：区块结构
merkle-tree.js
class Block {
constructor(prevHash, transactions, timestamp) {
this.header = {
prevHash,           // 前一区块哈希
merkleRoot: this.calcMerkleRoot(transactions),
timestamp,          // 时间戳
difficulty: 4,      // 难度（前导0个数）
nonce: 0            // 随机数
}
this.transactions = transactions  // 交易列表
this.hash = this.mine()  // 挖矿计算哈希
}

calcMerkleRoot(txs) {
// 计算Merkle根
const leaves = txs.map(tx => sha256(JSON.stringify(tx)))
return buildMerkleTree(leaves)
}

mine() {
// 工作量证明：找满足难度目标的哈希
let nonce = 0
let hash
const target = "0".repeat(this.header.difficulty)

do {
this.header.nonce = nonce
hash = sha256(JSON.stringify(this.header))
nonce++
} while (!hash.startsWith(target))

return hash
}
}

4. 链式结构：为什么无法篡改？
区块链之所以叫"链"，是因为每个区块都包含前一区块的哈希。
🔗  区块1 ← 创世区块  ↓
🔗  区块2 ← 包含区块1的哈希  ↓
🔗  区块3 ← 包含区块2的哈希  ↓
🔗  区块4 ← 包含区块3的哈希
篡改的后果：

假设你想篡改区块2的一笔交易：

1. 交易变了 → Merkle根变了 → 区块哈希变了
2. 区块2哈希变了 → 区块3的"前一哈希"对不上了
3. 你必须重新挖区块2、3、4...
4. 但网络只认最长的链，你算力不够，追不上

结论：篡改成本极高，几乎不可能
51%攻击
理论上，如果你能控制全网51%以上的算力，你可以：
制造一条更长的分叉链
回滚自己的交易（双花）
但成本极高：比特币网络算力相当于数百万台顶级矿机，攻击成本数十亿美元。
5. 总结

①  哈希函数是区块链的"数字指纹"，保证数据完整性
②  Merkle树让海量交易的验证变得高效
③  区块头包含前一区块哈希，形成链式结构
④  篡改一个区块需要重挖后续所有区块，成本极高
⑤  区块链的不可篡改性来自密码学 + 经济激励
