## 安全的本质：减少攻击面


钱包安全的核心不是某一个工具，而是减少攻击面——让攻击者能接触到你的资产的路径尽可能少。

攻击面分层模型：

🔑 私钥层：助记词泄露 = 全部归零
📱 设备层：电脑/手机被黑 = 签名被篡改
🌐 网络层：钓鱼/MITM = 骗你签恶意交易
👥 社交层：社工攻击 = 骗你主动转账

每一层都需要对应的防护措施。


## 硬件钱包：冷存储的核心


硬件钱包的核心原理：私钥永远不离开设备。你签名的交易在硬件设备内部完成，电脑只能看到签名结果，看不到私钥。

Ledger：Secure Element芯片（EAL5+认证），抗物理攻击能力强

Trezor：开源固件，透明度高，但使用普通芯片（无SE）

Keystone：二维码签名，气隔离——完全不存在USB连接被攻击的可能

contract.sol


```typescript
# 硬件钱包签名流程（Python伪代码）
from ledger_sdk import LedgerClient
from web3 import Web3
```


```bash
# 1. 连接硬件钱包
ledger = LedgerClient()
print('Ledger connected:', ledger.get_address())
```


```bash
# 2. 构造交易（在电脑上）
tx = {
'to': '0x1234...abcd',
'value': Web3.to_wei(1, 'ether'),
'gas': 21000,
'gasPrice': Web3.to_wei(30, 'gwei'),
'nonce': 0,
'chainId': 1,
}
```


```bash
# 3. 在硬件钱包上签名（私钥不离开设备）
signed_tx = ledger.sign_transaction(tx)
```


```bash
# 4. 广播签名后的交易
w3 = Web3(Web3.HTTPProvider("https://mainnet.infura.io/v3/YOUR_KEY"))
tx_hash = w3.eth.send_raw_transaction(signed_tx.rawTransaction)
print('TX sent:', tx_hash.hex())
```

## 多签钱包：不要把鸡蛋放一个篮子


多签钱包要求多个私钥共同签名才能执行交易。即使一个私钥被盗，攻击者也无法转走资金。

contract.sol


```javascript
// Gnosis Safe多签部署（JavaScript）
const { ethers } = require('ethers');
const Safe = require('@safe-global/protocol-kit');
```


```javascript
async function deployMultisig() {
// 3个签名者，需要2个签名（2-of-3）
const owners = [
'0xAAA...AAA',  // 你自己的地址
'0xBBB...BBB',  // 信任的家人/朋友
'0xCCC...CCC',  // 备用硬件钱包地址
];
const threshold = 2; // 需要2个签名
```


```javascript
const safeSdk = await Safe.init({
provider: 'https://mainnet.infura.io/v3/YOUR_KEY',
signer: PRIVATE_KEY,
safeAddress: await Safe.predictSafeAddress({
owners,
threshold,
}),
});
```


```javascript
const safeAddress = await safeSdk.getAddress();
console.log('Multisig deployed at:', safeAddress);
console.log('Owners:', owners);
console.log('Threshold:', threshold, 'of', owners.length);
}
```


```plaintext
deployMultisig();
deploy_multisig.js
```


多签阈值设置建议：

2-of-3：个人资产管理最常用。一把硬件钱包+一把备用+一个信任的人

3-of-5：团队/DAO资金管理。需要多数同意才能动用资金

避免1-of-N：等于没有多签，一个被盗就全完


## 助记词备份：最关键的一环


助记词是私钥的人类可读形式——12/24个英文单词可以恢复你所有的地址和资产。助记词泄露=资产归零。

contract.sol


```typescript
# Shamir Secret Sharing助记词分片（Python）
from shamir_mnemonic import generate_mnemonics
```


```bash
# 把24个助记词分成5份，需要3份才能恢复
mnemonic = "abandon abandon abandon ... about"  # your mnemonic
shares = generate_mnemonics(
master_secret=mnemonic.encode(),
threshold=3,      # 需要3份才能恢复
shares=5,         # 分成5份
)
```


```plaintext
for i, share in enumerate(shares):
print(f"Share {i+1}: {share}")
# 分别存放在5个不同地点
# 例如：银行保险箱/父母家/公司/信任的朋友/律师
```


```typescript
# 恢复时需要至少3份
from shamir_mnemonic import combine_mnemonics
recovered = combine_mnemonics(shares[:3])
print('Recovered:', recovered.decode())
```

## 冷热组合方案


最佳实践是冷热分离——日常用热钱包，大额用冷钱包。

热钱包（MetaMask/手机钱包）：放日常使用的资金，<10%总资产

温钱包（多签钱包）：放中等金额，需要多个签名才能动用

冷钱包（硬件钱包+保险箱）：放大额资产，平时不联网
