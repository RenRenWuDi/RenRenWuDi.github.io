智能合约入门
代码即法律
EVM、Solidity基础、部署你的第一个合约
💡 你将学到：智能合约概念、EVM原理、Solidity基础语法、部署流程
⏱ 阅读时间：约15分钟
1. 什么是智能合约？
智能合约（Smart Contract）这个概念最早由密码学家Nick Szabo在1994年提出。
智能合约是运行在区块链上的程序，当预设条件满足时自动执行。

就像自动售货机：投币 → 选商品 → 自动出货。不需要店员，代码自动执行。
智能合约的特点：

• 自动执行：条件触发，代码自动运行
• 不可篡改：部署后代码不可修改
• 透明公开：所有人都能查看代码逻辑
• 去信任：不需要第三方中介
"代码即法律"的含义
一旦智能合约部署到区块链，它就会严格按照代码逻辑执行，不受任何人控制。
这意味着：
合约写的是什么，执行的就是什么
即使代码有bug，也会按bug执行
没有"管理员"可以暂停或修改合约
⚠️ 警告：The DAO事件（2016年）就是因为智能合约漏洞，导致360万ETH被盗。这提醒我们：写合约必须极其谨慎，代码审计必不可少。
2. EVM：以太坊虚拟机
智能合约跑在哪里？答案是EVM（Ethereum Virtual Machine）。
EVM是什么？

• 以太坊的"操作系统"
• 每个节点都运行EVM
• 执行智能合约的字节码
• 保证所有节点执行结果一致
EVM执行流程
📝  开发者用Solidity编写合约  ↓
🔧  编译器将Solidity编译为EVM字节码  ↓
📡  部署交易广播到网络  ↓
⛏  矿工/验证者执行EVM字节码  ↓
💾  执行结果更新状态树
Gas机制
执行合约需要消耗计算资源，以太坊用Gas来计量。
Gas Limit：交易最多能用的Gas量
Gas Price：每单位Gas的价格（Gwei）
交易费 = Gas Used × Gas Price
SimpleStorage.sol
// 交易费用计算
const gasUsed = 21000           // 转账消耗的Gas
const gasPrice = 20             // 20 Gwei
const fee = gasUsed * gasPrice  // 420000 Gwei = 0.00042 ETH

// 复杂合约调用可能消耗几十万Gas
// Gas费防止无限循环攻击（跑不完就耗尽Gas）

3. Solidity：智能合约编程语言
Solidity是以太坊最常用的智能合约语言，语法类似JavaScript。
你的第一个合约
SimpleStorage.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
uint256 private storedData;

// 存储一个数字
function set(uint256 x) public {
storedData = x;
}

// 读取存储的数字
function get() public view returns (uint256) {
return storedData;
}
}

代码解析
SPDX-License-Identifier：开源许可证声明
pragma solidity ^0.8.0：指定编译器版本
contract：定义合约，类似class
uint256：无符号256位整数
public：函数可被外部调用
view：只读函数，不修改状态
关键概念
Storage vs Memory

• Storage：永久存储在区块链上（贵）
• Memory：临时存储，函数执行完释放（便宜）
• Calldata：函数参数，只读

默认情况下，状态变量存在Storage，局部变量存在Memory。
4. 部署你的第一个合约
部署合约也是一笔交易，只是to地址为空。
部署流程
1️⃣  编写Solidity代码  ↓
2️⃣  使用Remix或Hardhat编译  ↓
3️⃣  获取编译后的字节码和ABI  ↓
4️⃣  发送部署交易（data字段放字节码）  ↓
5️⃣  交易确认后，获得合约地址
代码示例：用Ethers.js部署
SimpleStorage.sol
const { ethers } = require('ethers')

// 连接网络（这里用测试网）
const provider = new ethers.JsonRpcProvider('https://goerli.infura.io/v3/YOUR_KEY')

// 加载钱包
const privateKey = '0x...'
const wallet = new ethers.Wallet(privateKey, provider)

// 合约字节码和ABI（从编译结果获取）
const bytecode = "0x60806040..."
const abi = [
"function set(uint256 x)",
"function get() view returns (uint256)"
]

// 部署合约
async function deploy() {
const factory = new ethers.ContractFactory(abi, bytecode, wallet)
const contract = await factory.deploy()
await contract.waitForDeployment()

console.log('合约地址:', await contract.getAddress())
}

deploy()

5. 与合约交互
部署后，任何人都可以调用合约的public函数。
两种调用方式
Transaction（交易）

• 修改合约状态的函数
• 需要签名、支付Gas
• 被打包进区块后才生效

Call（调用）

• 只读函数（view/pure）
• 不需要签名，不消耗Gas
• 本地执行，立即返回结果
代码示例：调用合约
SimpleStorage.sol
const { ethers } = require('ethers')

// 已部署的合约地址
const contractAddress = '0x123...'

// 创建合约实例（只读，不需要私钥）
const contract = new ethers.Contract(contractAddress, abi, provider)

// 调用view函数（免费）
async function readData() {
const value = await contract.get()
console.log("存储的值:", value.toString())
}

// 发送交易修改状态（需要私钥）
async function writeData() {
const contractWithSigner = contract.connect(wallet)
const tx = await contractWithSigner.set(42)
await tx.wait()
console.log("交易已确认")
}

6. 智能合约安全：避坑指南
智能合约一旦部署就无法修改，安全至关重要。
常见漏洞
重入攻击：合约在转账时被递归调用
整数溢出：数值计算超出范围
访问控制：敏感函数没有权限检查
随机数问题：区块链上无法生成真随机数
安全最佳实践
使用Checks-Effects-Interactions模式：先检查条件，再修改状态，最后交互
使用SafeMath库防止溢出（Solidity 0.8+已内置）
敏感函数加onlyOwner修饰符
部署前进行专业审计
考虑使用代理合约模式，允许升级
7. 总结

①  智能合约是运行在区块链上的自动执行程序
②  EVM是以太坊的执行环境，Gas机制防止资源滥用
③  Solidity是智能合约的主要编程语言
④  部署合约需要编译、发送交易、获得合约地址
⑤  合约安全至关重要，审计和最佳实践必不可少
