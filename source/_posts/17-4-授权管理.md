## 用Revoke.cash守护你的钱包——ERC-20授权的坑与对策


## 授权机制回顾：为什么需要管理？


ERC-20的approve机制是DeFi的基础——你授权合约使用你的Token，合约才能帮你做Swap、做LP、做借贷。但授权是双刃剑：授权的合约如果被黑或恶意，可以转走你授权额度的所有Token。

真实案例：

2022年某DeFi协议被黑，攻击者利用合约的漏洞
转走了所有授权用户的Token——
不是用户的私钥被盗，而是用户之前授权过这个合约。

你的授权 = 合约可以动用你的资金
授权管理 = 限制合约能动用多少、多久


## 查看你的授权


每次你连接钱包到DeFi网站，可能都会留下授权。这些授权不会自动过期——除非你主动撤销。用Revoke.cash可以查看所有授权：

contract.sol


```javascript
// 用ethers.js查询你的所有授权（JavaScript）
const { ethers } = require('ethers');
```


```javascript
const provider = new ethers.JsonRpcProvider(
'https://mainnet.infura.io/v3/YOUR_KEY'
);
```


```javascript
const ERC20_ABI = [
"function allowance(address owner, address spender) view returns (uint256)",
"function approve(address spender, uint256 amount) returns (bool)",
'function name() view returns (string)',
'function symbol() view returns (string)',
];
```


```javascript
async function checkAllAllowances(ownerAddress, tokenAddresses) {
for (const tokenAddr of tokenAddresses) {
const token = new ethers.Contract(tokenAddr, ERC20_ABI, provider);
const name = await token.name();
// 查询常见spender地址（已知的DeFi协议）
const spenders = [
'0x68b3465833fb72A70ecDF485E0e4C7bD8665Fc45', // Uniswap Router
'0xE592427A0AEce92De3Edee1F18E0157C05861564', // Uniswap V3 Router
'0xDef1C0ded9bec7F1a1670819833240f027b25EfF', // 0x Exchange
];
for (const spender of spenders) {
const allowance = await token.allowance(ownerAddress, spender);
if (allowance > 0n) {
console.log(`${name}: approved ${spender} for ${ethers.formatUnits(allowance, 18)}`);
}
}
}
}
```

## 撤销授权


发现异常或不需要的授权后，立即撤销：

contract.sol


```javascript
// 撤销ERC-20授权（JavaScript）
async function revokeAllowance(tokenAddress, spenderAddress) {
const token = new ethers.Contract(tokenAddress, ERC20_ABI, wallet);
// 将授权额度设为0
const tx = await token.approve(spenderAddress, 0);
console.log('Revoke tx sent:', tx.hash);
await tx.wait();
console.log('Revoke confirmed! Allowance set to 0.');
}
```


```javascript
// 批量撤销所有授权
async function revokeAll(tokenAddress, spenderAddresses) {
for (const spender of spenderAddresses) {
await revokeAllowance(tokenAddress, spender);
}
console.log('All allowances revoked!');
}
```

## 安全授权的最佳实践


原则一：最小授权——只授权你需要使用的金额，不要用type(uint256).max无限授权

原则二：及时撤销——用完DeFi协议后，立即撤销授权

原则三：定期检查——每周用Revoke.cash检查一次授权列表

原则四：只用知名协议——小协议被黑的概率更高

contract.sol


```javascript
// 安全授权工具：精确控制授权金额（JavaScript）
async function safeApprove(tokenAddress, spender, amount) {
const token = new ethers.Contract(tokenAddress, ERC20_ABI, wallet);
// 1. 检查当前授权
const current = await token.allowance(wallet.address, spender);
if (current > 0n) {
console.log('Current allowance:', ethers.formatUnits(current, 18));
console.log('Revoking old allowance first...');
await token.approve(spender, 0n);
await ethers.sleep(1000);
}
// 2. 设置精确的授权金额
console.log('Approving exact amount...');
const tx = await token.approve(spender, amount);
await tx.wait();
console.log('Approved:', ethers.formatUnits(amount, 18));
}
```

## 自动化授权监控


用脚本定期监控授权变化，发现异常立即告警：

contract.sol


```typescript
# 授权监控脚本（Python）
import requests
import json
from datetime import datetime
```


```python
def monitor_allowances(address, webhook_url=None):
"""监控地址的所有ERC-20授权"""
# 用Etherscan API获取所有approve事件
api_key = "YOUR_ETHERSCAN_KEY"
url = f"https://api.etherscan.io/api?module=account&action=tokentx&address={address}&apikey={api_key}"
response = requests.get(url)
txs = response.json()["result"]
approvals = []
for tx in txs:
if tx.get("input", "").startswith("0x095ea7b3"):  # approve函数签名
spender = "0x" + tx["input"][34:74]
amount = int(tx["input"][74:138], 16)
approvals.append({
"token": tx["contractAddress"],
"spender": spender,
"amount": amount,
"timestamp": datetime.fromtimestamp(int(tx["timeStamp"])).isoformat(),
})
# 检查是否有新的无限授权
for a in approvals:
if a["amount"] >= 2**255:  # 接近type(uint256).max
alert = f"⚠️ INFINITE APPROVAL DETECTED!\nToken: {a['token']}\nSpender: {a['spender']}\nTime: {a['timestamp']}"
print(alert)
if webhook_url:
requests.post(webhook_url, json={"text": alert})
return approvals
```
