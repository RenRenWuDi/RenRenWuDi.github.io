## DAO与治理机制


## 去中心化自治组织的艺术


## DAO是什么？


DAO（去中心化自治组织）是用代码治理的组织。没有CEO，没有董事会，所有规则写在智能合约里，重大决策由Token持有者投票决定。

DAO的核心特征：

* 无中心化控制：规则由代码执行
* Token持有者治理：1 token = 1 vote（或加权）
* 公开透明：所有提案和投票链上可查
* 自动执行：投票通过后合约自动执行


## DAO vs 传统公司


传统公司靠法律合同约束，DAO靠智能合约约束。传统公司的"公司章程"是PDF文件，DAO的"公司章程"是代码。

DAO vs 传统公司的关键区别：

* 准入：任何人持有Token即可参与
* 决策：链上投票取代董事会
* 执行：合约自动执行取代人工
* 透明度：所有操作公开透明


## DAO的类型


**协议DAO：Uniswap、Compound，治理协议本身**

**投资DAO：BitDAO、MetaCartel，DAO做VC投资**

**社交DAO：FWB（Frien with Benefits），围绕社交目的**

**收藏DAO：PleasrDAO，购买NFT艺术**

**服务DAO：RaidGuild，提供外包服务**


## Token治理模型


最常见的DAO治理是Token投票。持有治理Token就有投票权，但Token集中度是大问题——创始团队可能持有大量Token。

Token治理的挑战：

* Token集中：大户/团队可能控制投票
* 投票率低：很多成员不投票
* 时间锁延迟：恶意提案通过后有时间窗口取消
* 提案门槛：需要有足够Token才能发起提案


## Governor合约


OpenZeppelin的Governor是标准的DAO治理合约，支持提案、投票、执行全流程。

Governor.sol


```solidity
import "@openzeppelin/contracts/governance/Governor.sol";
import "@openzeppelin/contracts/governance/extensions/GovernorCountingSimple.sol";
contract MyDAO is Governor, GovernorCountingSimple {
uint public votingDelay = 1 days;
uint public votingPeriod = 7 days;
uint public proposalThreshold = 1000e18;
function propose(address[] memory targets, uint256[] memory values,
bytes[] memory calldatas, string memory description)
public override returns (uint256)
{
return super.propose(targets, values, calldatas, description);
}
function vote(uint256 proposalId, uint8 support)
external override
{
return _vote(msg.sender, proposalId, support);
}
}
```


## Quadratic Funding（二次方融资）


Gitcoin Grants使用Quadratic Funding（QF）。不是谁投钱多谁说了算，而是小额捐款者数量越多匹配资金越多——让普通用户的声音被放大。

QF公式：

* 政府/DAO匹配资金 = 匹配池 * (sqrt(小捐款者数)求和)^2
* 100个人捐1元 = 100元^2 = 10000元匹配
* 1个人捐100元 = 10元^2 = 100元匹配
* 结论：小额多数 > 大额少数


## 真实DAO案例


MakerDAO：稳定币DAI的治理DAO。持有MKR Token可以投票决定利率、抵押品类型等参数。是历史最悠久的DeFi DAO之一。

Compound：Compound Finance的治理DAO。治理参数包括利率模型、清算阈值等。但2023年因治理攻击导致COMP价格暴跌的教训。


## DAO的安全风险


治理攻击：大户通过借贷获得大量Token进行攻击

时间锁延迟：恶意提案通过后需要时间窗口执行

提案门槛过低：容易发起垃圾提案

投票率低：少数人控制结果

**9. 总结**

①  DAO = 代码治理的组织，无中心化控制

②  Token投票是最常见的DAO治理方式

③  Governor合约是OpenZeppelin的标准DAO合约

④  Quadratic Funding让小额多数声音被放大

⑤  DAO面临Token集中、投票率低等现实问题
