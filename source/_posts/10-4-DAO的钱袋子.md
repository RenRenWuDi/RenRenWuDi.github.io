## DAO国库到底是管什么？


DAO国库（Treasure）是协议运作的"钱袋子"。它存放着Token授权、协议收入留存和战略资金。怎么管这笔钱，是治理里最敏感的话题——既要让团队有足够的钱可用，又要防止权力被滥用。

说白了，国库管理要回答三个问题：谁来动钱、花多少、怎么监督。


## 多签钱包：最基础也是最重要的工具


多签钱包（Multi-Sig Wallet）是最常见的国库管理工具。一笔转账需要多个私钥签名才能执行，这样即使一个签名者的私钥泄露，攻击者也无法单独转移资金。

多签的核心参数：签名者数量 N，和通过门槛 M（M-of-N）。例如 3/5 表示：5个签名者中，至少需要3人同意才能执行交易。

以太坊上最流行的多签钱包是Safe（原Gnosis Safe），支持任意 M-of-N 配置。MakerDAO 用的是 5/9 多签，Uniswap 社区用的是 4/7。

如果你想快速搭建一个多签合约，可以用OpenZeppelin的SDK。但作为学习目的，我们来看一下简化版的实现原理：

SimpleMultiSig.sol


```solidity
contract SimpleMultiSig {
address[] public signers;          // 签名者列表
uint256 public threshold;          // 门槛：至少需要几个人同意
uint256 public txnCount;           // 已执行交易计数
```


```solidity
struct Transaction {
address to;
uint256 value;
bytes data;
bool executed;
uint256 confirmations;
}
```


```solidity
mapping(uint256 => mapping(address => bool)) confirmed;
Transaction[] public transactions;
```


```javascript
constructor(address[] memory _signers, uint256 _threshold) {
require(_signers.length >= _threshold, "Invalid threshold");
signers = _signers;
threshold = _threshold;
}
```


```javascript
function submitTransaction(address to, uint256 value, bytes memory data) external {
require(isSigner(msg.sender), "Not a signer");
uint256 id = transactions.length;
transactions.push(Transaction({
to: to, value: value, data: data,
executed: false, confirmations: 0
}));
confirm(id); // 提案人默认投赞成票
}
```


```javascript
function confirm(uint256 id) public {
require(isSigner(msg.sender), "Not a signer");
require(!confirmed[id][msg.sender], "Already confirmed");
confirmed[id][msg.sender] = true;
transactions[id].confirmations++;
if (transactions[id].confirmations >= threshold) {
execute(id);
}
}
```


```javascript
function execute(uint256 id) internal {
Transaction storage txn = transactions[id];
require(!txn.executed, "Already executed");
(bool ok,) = txn.to.call{value: txn.value}(txn.data);
require(ok, "Execution failed");
txn.executed = true;
}
```


```javascript
function isSigner(address a) internal view returns(bool){
for(uint256 i=0;i<signers.length;i++)
if(signers[i]==a) return true;
return false;
}
}
```


这个简化版展示了多签的核心逻辑：收集签名 → 达到门槛 → 执行交易。实际生产中建议用OpenZeppelin的Governor+Timelock组合，更安全且经过审计。


## 预算审批链：钱怎么花出去？


多签解决了"谁来签"的问题，但"谁可以申请、花多少"同样重要。常见的几种国库支出模式：

直接治理审批：每笔支出都上链投票，适合小额、频繁的支出（<1万美元）

预算委员会：社区投票通过季度预算（如每月10万美元），委员会在预算内自主支配

分级审批：小额（如<1 ETH）直接多签，大额走完整治理提案

预算委员会适合什么样的DAO？

团队规模较大、日常运营支出多（例如工资、服务器、赏金）。Compound和Aave都采用这种模式：社区按季度审批运营预算，团队在预算内自主决定怎么花。

MakerDAO则更偏向直接治理——超过一定金额的支出，必须走完整提案+投票流程。他们有专门的"核心预算"和"生态系统预算"两条线，分开管理。


## 时间锁：给国库加上冷却期


多签+时间锁是国库安全的黄金组合。提案通过后，资金不会立即转出，而是进入一个等待期（通常2-48小时），这段时间内：

社区可以审查即将执行的交易

如果发现异常（如大额转账到未知地址），可以撤离相关资产

给第三方系统（预言机、跨链桥）留出适配时间

TimeLock.sol


```solidity
// 时间锁合约的简化逻辑
contract TimeLock {
uint256 public delay = 2 days;  // 等待2天
```


```solidity
mapping(bytes32 => uint256) public queued;
```


```javascript
function queue(address to, uint256 value, bytes calldata data)
external returns(bytes32)
{
bytes32 id = keccak256(abi.encode(to, value, data));
queued[id] = block.timestamp + delay;
return id;
}
```


```javascript
function execute(address to, uint256 value, bytes calldata data)
external payable
{
bytes32 id = keccak256(abi.encode(to, value, data));
require(queued[id] != 0, "Not queued");
require(block.timestamp >= queued[id], "Too early");
queued[id] = 0;
(bool ok,) = to.call{value: value}(data);
require(ok, "Execute failed");
}
}
```


## 透明度：让社区看得见每一笔钱


国库透明度是社区信任的根基。哪怕是去中心化的协议，如果国库不透明，社区就没法真正监督治理。

链上公开国库地址：用Etherscan或Debank随时查余额和交易历史

定期发布财务报告：收入来源、支出明细、余额变化，MakerDAO每月发一次"财务健康报告"

使用追踪工具：Llama、DeepDAO可以聚合多链国库数据，做可视化分析


## 国库投资的边界


有些DAO不满足于"存钱"，还想让国库增值。常见策略：

Token分散：把一部分治理Token换成稳定币，降低波动风险

收益聚合：闲置资金放入Uniswap V3做LP或Lido质押，赚点被动收入

战略投资：用国库投资生态内早期项目，换取Token激励或股权

国库投资最容易踩的坑：

投资决策本身没有经过治理，团队私下操作亏损了怎么办？所以MakerDAO和Bitdao都明确规定：超过一定金额的投资必须走治理提案，不能先斩后奏。
