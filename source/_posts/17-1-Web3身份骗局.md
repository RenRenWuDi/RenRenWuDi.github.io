## Web3里的"身份骗局"


## 签名授权那些坑——从假网站到恶意授权的识别与防御


## Web3身份骗局和传统骗局的区别


传统骗局偷的是密码——你输入账号密码，对方拿去登录。但Web3骗局偷的不是密码，是授权（Approval）。你不需要输入任何东西，只要点一个"连接钱包"按钮，签一个名，你的资产就没了。

传统骗局：骗密码 -> 登录你的账号 -> 转走资金

Web3骗局：骗签名 -> 直接授权转账 -> 资产归零

区别：Web3骗局不需要你输入任何敏感信息，
只需要你签一个名——而这个签名本身是合法的区块链交易。

这就是为什么Web3签名骗局比传统骗局更危险——你的硬件钱包也救不了你，因为签名是你自己授权的。


## 七种常见签名骗局手法


手法一：假网站仿冒——克隆Uniswap/1inch等知名DeFi网站，诱导用户连接钱包并签名

手法二：空投诱导——发送假的Token到你的地址，附带一个claim链接，点进去就触发恶意授权

手法三：社交工程仿冒——冒充官方客服，发私信给你一个"解决问题"的链接

手法四：NFT诱导——空投一个NFT到你的地址，NFT的图片上有claim网址，诱导你访问

手法五：permit签名陷阱——利用EIP-2612 permit，不需要授权交易，一个签名就能转走你的Token

手法六：permit2批量授权陷阱——利用permit2，一次签名授权所有Token

手法七：钱包界面仿真——伪造MetaMask弹窗，诱导用户在假弹窗中输入助记词


## 恶意授权的原理


要防范签名骗局，首先要理解授权（Approval）机制。ERC-20标准中，approve函数允许你授权某个地址使用你的Token。骗局的本质就是诱导你调用approve，授权给恶意地址。

contract.sol


```solidity
// ERC-20 approve机制——签名骗局的核心靶点
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
```


```solidity
interface IERC20 {
function approve(address spender, uint256 amount) external returns (bool);
function transferFrom(address from, address to, uint256 amount) external returns (bool);
function allowance(address owner, address spender) external view returns (uint256);
}
```


```solidity
contract MaliciousExample {
// 恶意合约：诱导用户调用这个函数
function fakeClaim() external {
// 用户以为是领取空投
// 实际上是授权恶意地址转走所有Token
IERC20(0xUSDC).approve(msg.sender, type(uint256).max);
// ↑ 这行让msg.sender（恶意方控制的合约）
//   可以转走你所有的USDC
}
}
```



## permit签名陷阱：更隐蔽的威胁


传统approve需要在链上发一笔交易，会消耗Gas。但EIP-2612引入了permit函数——你只需要签一个链下签名，别人就可以拿着这个签名去链上帮你做approve。这意味着你不需要花Gas费，不需要发交易，只需要签个名——你的Token就被授权了。

```javascript
// EIP-2612 permit签名检测脚本（JavaScript）
```

```javascript
return 'SAFE';
} catch(e) {
```

## 如何识别仿冒网站


检查URL拼写：uniswap.org正确，uniswaap.org是仿冒。用书签代替手动输入URL

检查SSL证书：仿冒网站往往没有有效SSL证书，或者证书域名不匹配

用Etherscan验证：在签名前，把合约地址放到Etherscan上查——看是不是已知协议

用Revoke.cash检查：定期查看你有哪些授权，发现异常立即撤销

用PocketUniverse/Rabby等安全插件：它们会在你签名前分析交易内容
