## 链上数据为什么重要？


链上数据是Web3的"真相来源"。不像传统公司的数据藏在服务器里，区块链上的每一笔交易都是公开的。懂链上数据，你就有了信息优势。

链上数据的价值：

* 投资决策：追踪巨鲸钱包、庄家动向
* 项目调研：看真实用户数、TVL、交易量
* 竞品分析：对比各协议的活跃度
* 市场营销：量化增长指标


## Dune Analytics是什么？


Dune是链上数据的Google。你可以用SQL查询以太坊、Solana等链上的交易数据，然后做成漂亮的可视化面板。

Dune的核心概念：

* Tables：链上数据表（如ethereum.transfers）
* Decoded Contracts：解码后的合约（如Uniswap V2）
* Queries：SQL查询
* Visualizations：可视化图表
* Dashboards：面板整合


## Dune数据表结构


Dune的数据分几类：raw data（原始事件）、decoded data（解码后的合约调用）、spells（聚合表）。

常用表：

* ethereum.transactions：所有交易
* ethereum.logs：所有事件日志
* ethereum.traces：内部调用
* erc20.Transfer：ERC20转账事件
* erc721.Transfer：NFT转账事件


## SQL查询示例


以下SQL查询Uniswap V2的日交易量和活跃地址数。

SQL


```typescript
SELECT
date_trunc('day', evt_block_time) as day,
COUNT(DISTINCT tx_from) as users,
COUNT(*) as swaps,
SUM(token_a_amount + token_b_amount) as volume
FROM uniswap_v2."SwapEvent"
WHERE evt_block_time > NOW() - INTERVAL '30 days'
GROUP BY 1
ORDER BY 1 DESC
;
SwapVolume.sql
```


## 追踪巨鲸钱包


SQL


```typescript
WITH whale_tx AS (
SELECT
from AS wallet,
to AS contract,
value / 1e18 as eth_amount,
block_time
FROM ethereum.transactions t
WHERE value > 1000 * 1e18
AND block_time > NOW() - INTERVAL '7 days'
AND trace_address = array[]
SELECT
wallet,
SUM(eth_amount) as total_in,
COUNT(*) as tx_count
FROM whale_tx
GROUP BY wallet
ORDER BY total_in DESC
LIMIT 20
;
WhaleTracker.sql
```
