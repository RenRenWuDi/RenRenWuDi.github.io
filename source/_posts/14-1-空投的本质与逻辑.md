## 空投不赚钱就不会有项目发


空投表面上是项目方给早期用户的"免费午餐"。但经济学的第一条准则是：没有免费午餐。项目发空投一定有明确的商业动机——而且这个动机和"慈善"没有任何关系。

项目方发空投的三大动机：\n\n1. 代币分发去中心化：大量用户持有代币，治理更去中心化（ERC-20分发）\n2. 市场推广与冷启动：空投制造话题和关注度，替代传统营销预算\n3. 合规需求：空投作为"用服务换Token"的法律框架——美国SEC更可能认定用户通过劳动获得代币，而不是投资

对用户来说，空投本质上是"用你的注意力和交互行为换取未来的代币"。你的钱包地址、交互频次、资金规模——这些都是你在"链上信誉"的证明。


## 空投的筛选标准


项目方如何判定谁"值得"拿空投？大致分四类筛选标准：

交互深度（最核心）：交易次数、交互合约数量、使用产品功能的广度——不是"点了几下"而是"真正用了"

资金规模：存入的TVL、交易金额、持有资产种类——越大的资金投入说明你越认真参与

时间维度：交互的时间跨度、持续活跃度——项目方不要"一夜情"用户

参与者粘性：是否参与治理、是否加入社区Discord、是否推荐朋友

```python
def airdrop_score(interactions, tvl, months_active, referrals):
score = 0
score += min(interactions * 5, 300)  # 交互深度上限
score += min(tvl * 0.1, 200)         # 资金上限
score += min(months_active * 20, 100) # 粘性上限
score += min(referrals * 10, 50)      # 推荐上限
return score
```

## 女巫攻击：薅羊毛的"规模化"


女巫攻击（Sybil Attack）是空投生态里的"影子战术"——一个人伪造成几千个独立用户来领取空投。

女巫攻击的基础手段：\n\n1. 批量生成钱包地址：一个私钥控制数百个地址\n2. 资金分发：从主钱包分ETH/NFT到各个地址\n3. 模拟交互：每个地址执行类似的交互模式\n\n高级女巫：用不同的IP、不同的浏览器指纹、不同的时间窗口做"类真实"交互

女巫攻击的成本：每个地址需要至少0.01-0.1 ETH的Gas费。如果一个项目空投价值5000万，每个地址分200U，25万个地址——女巫方需要投入约250-2500ETH的Gas成本。高Gas链上，女巫的经济账可能不划算。


## 项目方的反女巫手段


项目方不是傻子，他们也在不断升级反女巫手段：

链上行为分析：跨地址转账模式、Gas消耗模式、交互时间分布——AI模型可以识别"类女巫"模式

身份验证：Gitcoin Passport、World ID——要求用户证明自己是真人

资金溯源：跟踪ETH来源是否反复从同一个CEX提币到多个地址

社交验证：Discord身份关联、Twitter交叉验证

反女巫最关键的一招：\n\n高阈值筛选 + 真金白银的Gas消耗。如果一个交互操作需要多次交易、高Gas消耗，女巫成本会急剧上升，女巫攻击的经济模型就会不划算。\n\n2023年的Arbitrum空投和2024年的Starknet空投都用了大力度的反女巫算法，筛掉了大量女巫地址。


## 空投经济学：鱼与熊掌


好的空投既要分发广，又要把代币给到"正确的人"。这两个目标本身就是矛盾的：

分发广 → 门槛低 → 容易被女巫大量薅走

筛得严 → 门槛高 → 真正的早期用户可能被误伤

Uniswap选择了门槛极低（仅需一次交互），结果空投价值被稀释。Starknet选择了高门槛，结果大量真实用户被误判为女巫。没有任何方案是完美的。
