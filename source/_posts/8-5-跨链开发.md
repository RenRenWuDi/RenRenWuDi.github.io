**跨链开发**


## LayerZero与消息传递


## 为什么需要跨链？


区块链是多链世界。以太坊有DeFi，Solana有高性能应用，Cosmos有应用链，比特币是价值存储。资产和数据在这些链之间流动，就需要跨链协议。

跨链的核心需求：

* 资产跨链：把代币从一条链移到另一条
* 消息跨链：让合约跨链通信
* 状态同步：读取其他链的状态
* 互操作：调用其他链的合约


## LayerZero是什么？


LayerZero是全链互操作协议（Omnichain Interoperability Protocol）。它让合约可以在任意链之间发送消息，实现真正的"一次部署，全链可用"。

LayerZero核心组件：

* Endpoint：每条链上的入口合约
* Oracle：提供链的区块头
* Relayer：传递消息和证明
* ULN（User Application Network）：验证层


## LayerZero工作原理


LayerZero用Oracle和Relayer两个独立角色来保证安全。Oracle提供源链的状态证明，Relayer传递消息。只有两者一致，消息才被接受。

LayerZero消息流程：

1. 源链合约调用Endpoint.send()
2. 触发事件，包含消息和目标链信息
3. Oracle提供源链区块头
4. Relayer传递消息和证明
5. 目标链Endpoint验证
6. 调用目标合约的lzReceive()


## LayerZero合约示例


CrossChainToken.sol


```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
import "@layerzerolabs/contracts/LzApp.sol";
contract CrossChainToken is LzApp {
string public name = "CrossChainToken";
mapping(address => uint256) public balanceOf;
constructor(address _endpoint) LzApp(_endpoint) {}
function mint(address to, uint256 amount) external {
balanceOf[to] += amount;
}
function crossChainTransfer(uint16 dstChainId, address to, uint256 amount) external payable {
require(balanceOf[msg.sender] >= amount);
balanceOf[msg.sender] -= amount;
bytes memory payload = abi.encode(to, amount);
_lzSend(dstChainId, payload, payable(msg.sender), address(0), "", msg.value);
}
function _nonblockingLzReceive(uint16, bytes memory, bytes memory payload) internal override {
(address to, uint256 amount) = abi.decode(payload, (address, uint256));
balanceOf[to] += amount;
}
}
```


## 全链应用（Omnichain App）


全链应用是一次部署到多条链，状态在各链间同步。用户可以在任意链上操作，体验一致。

全链应用的优势：

* 统一流动性：一条链的流动性全链可用
* 最佳路由：自动选择最优路径
* 用户体验：无需关心在哪条链
* 组合性：与其他全链协议无缝组合


## Stargate：LayerZero上的跨链DEX


Stargate是LayerZero上的跨链原生资产桥。它实现了统一流动性池，用户可以在任意链Swap到任意链的资产。

Stargate特点：

* 统一流动性：所有链共享流动性池
* 即时确认：无需等待源链确认
* 保证交付：消息一定送达
* 低滑点：大流动性降低滑点


## 跨链安全考量


跨链安全风险：

* Oracle作恶：提供虚假状态
* Relayer作恶：传递虚假消息
* 双重花费：资产在两条链上同时存在
* 代码漏洞：跨链合约被攻击

LayerZero通过独立Oracle和Relayer降低风险：只有两者合谋才能攻击，而它们由不同实体运营。
