## 智能合约审计入门


## 从读代码到用Slither——自己动手做安全审计


## 为什么要学审计？


你不需要成为专业审计师，但如果你在DeFi里投钱，你至少要能看懂合约代码在做什么。很多rug pull和漏洞利用，只要稍微读一下代码就能发现。

投资决策中的审计层级：

第一层：看项目方有没有请审计公司（最低要求）
第二层：自己看审计报告，理解风险点
第三层：自己读合约代码，验证审计报告
第四层：自己跑Slither，找漏洞

你能做到第几层，决定了你能避开多少坑。


## 读合约代码的五个要点


拿到一个合约，不需要逐行读懂，重点看这五个地方：

1. 权限控制：谁可以调用什么函数？有没有onlyOwner？有没有timelock？

2. 资金流向：钱从哪进、到哪出？有没有提款函数？提款条件是什么？

3. 外部调用：合约调用了哪些外部合约？有没有重入风险？

4. 升级机制：合约是可升级的？升级权限在谁手里？

5. 紧急开关：有没有pause函数？谁可以触发？


## 用Slither做自动化审计


Slither是Trail of Bits开发的Solidity静态分析工具。它不需要运行合约，只通过分析代码就能发现潜在漏洞。

contract.sol


```bash
# 安装Slither（需要Python3和solc）
pip3 install slither-analyzer
```


```bash
# 或者用pipx安装（推荐）
pipx install slither-analyzer
```


```bash
# 对单个合约文件运行审计
slither contracts/Token.sol
```


```bash
# 对整个项目运行审计
slither .
```


```bash
# 只输出高 severity 的问题
slither . --high-severity
```


```bash
# 输出JSON格式（适合自动化处理）
slither . --json slither-report.json
```


**`install_slither.sh`**


Slither能检测的问题包括：重入漏洞、整数溢出、未检查的call返回值、权限控制问题、危险的标准库使用等。


## 实战：审计一个有漏洞的合约


下面是一个典型的有漏洞的合约，我们用Slither和手动审查来发现问题：

contract.sol


```solidity
// 有漏洞的合约——找出所有问题
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;
```


```solidity
contract VulnerableVault {
mapping(address => uint256) public balances;
address public owner;
IERC20 public token;
```


```solidity
constructor(address _token) {
owner = msg.sender;
token = IERC20(_token);
}
```


```javascript
// 漏洞1: 重入漏洞
function withdraw() external {
uint256 bal = balances[msg.sender];
require(bal > 0, "No balance");
// ❌ 先转账再清零 -> 重入漏洞！
(bool success,) = msg.sender.call{value: bal}("");
require(success, "Transfer failed");
balances[msg.sender] = 0; // 应该在call之前清零
}
```


```javascript
// 漏洞2: 无权限控制的提款
function emergencyWithdraw() external {
// ❌ 任何人都能调用！
token.transfer(msg.sender, token.balanceOf(address(this)));
}
```


```javascript
// 漏洞3: owner可以无限铸造
function mint(address to, uint256 amount) external {
// ❌ 没有onlyOwner！
balances[to] += amount;
}
```


```javascript
// 漏洞4: 未检查外部调用返回值
function deposit(uint256 amount) external {
// ❌ 没有检查transfer返回值
token.transferFrom(msg.sender, address(this), amount);
balances[msg.sender] += amount;
}
}
VulnerableVault.sol
```

## 修复漏洞


修复后的安全版本：

contract.sol


```solidity
// 修复后的安全合约
contract SafeVault {
mapping(address => uint256) public balances;
address public owner;
IERC20 public token;
bool public paused;
```


```javascript
modifier onlyOwner() {
require(msg.sender == owner, "Not owner");
_;
}
modifier notPaused() {
require(!paused, "Paused");
_;
}
```


```javascript
// CEI模式：先检查，再修改状态，最后交互
function withdraw() external notPaused {
uint256 bal = balances[msg.sender];
require(bal > 0, "No balance");
balances[msg.sender] = 0; // ✅ 先清零（Checks-Effects-Interactions）
(bool success,) = msg.sender.call{value: bal}("");
require(success, "Transfer failed");
}
```


```javascript
function emergencyWithdraw() external onlyOwner {
require(paused, "Not paused"); // ✅ 只有pause后才能用
token.transfer(owner, token.balanceOf(address(this)));
}
```


```javascript
function mint(address to, uint256 amount) external onlyOwner { // ✅ 加权限
balances[to] += amount;
}
```


```javascript
function deposit(uint256 amount) external notPaused {
require(token.transferFrom(msg.sender, address(this), amount), "Transfer failed"); // ✅ 检查返回值
balances[msg.sender] += amount;
}
```


```javascript
function setPaused(bool _paused) external onlyOwner {
paused = _paused;
}
}
SafeVault.sol
```
