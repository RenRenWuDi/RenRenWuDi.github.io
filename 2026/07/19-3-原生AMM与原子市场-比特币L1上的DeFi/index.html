原生AMM与原子市场：比特币L1上的DeFi

发表于2026-07-13|更新于2026-07-14|隐私DeFi
|总字数:2k|阅读时长:7分钟|浏览量:

比特币L1上的自动做市商：均匀出清+批量撮合+无MEV

1. 在比特币上做AMM有多难

在以太坊上做AMM（如Uniswap）很自然：合约持有资金池，用户调用swap函数，合约自动计算价格并完成交换。但比特币没有图灵完备的VM——你不能部署一个”自动撮合合约”。

tacit的方案是：AMM逻辑由索引器执行，资金不锁在合约里而是由索引器跟踪。核心创新是”均匀出清价格”（Uniform Clearing Price）和”区块批量撮合”。

2. 均匀出清价格（UCP）

传统AMM（如Uniswap V2）使用恒定乘积公式 x*y=k，每笔交易都会移动价格曲线。这意味着同一区块内，先交易的人获得更好的价格，后交易的人承受更大滑点——这恰恰是MEV的温床。

tacit的AMM不用恒定乘积。它采用均匀出清价格（UCP）：

UCP的核心思想：

  1. 一个区块内所有同方向的交易（比如都用A换B）被打包成一批
  2. 系统计算一个统一的出清价格，使总供给=总需求
  3. 同一批次内所有交易者获得相同价格

不存在”先到先得”——同一区块内第一笔和最后一笔交易价格完全一样。

具体来说，UCP通过匹配供需曲线的交点来确定价格。假设一个区块内有3笔用A换B的交易，分别想换10、20、15个A：

总供给：45个A（想换成B的交易者提供）

系统找到价格P，使得在价格P下，愿意用B换A的需求恰好等于45个A

所有3笔交易都按价格P成交

这个机制的关键特性是：交易者无法通过抢先交易（front-running）获得更好价格。

3. 区块批量撮合

tacit的AMM不是逐笔撮合，而是按区块批量撮合。流程如下：

批量撮合流程：

  1. 交易者在区块N内提交交易（包含想交易的资产和方向）
  2. 索引器收集区块N内所有交易
  3. 索引器按交易对分组，计算每对的UCP
  4. 在区块N+1开始时，所有交易按UCP成交
  5. 索引器更新池子储备和交易者余额

注意：成交发生在下一个区块，不是当前区块。这给了索引器足够时间计算。

池子储备是公开的——索引器跟踪并公开每个池的总量。但每个交易者的具体交易金额是机密的（用Pedersen承诺隐藏）。

4. 为什么”构造上无MEV”

MEV（最大可提取价值）在以太坊上是一个大问题：验证者可以重新排序、插入或删除交易来获利。tacit的AMM从构造上消除了MEV：

无抢先交易：同一区块内所有交易同价，先提交没有优势

无三明治攻击：没有逐笔价格滑动，三明治攻击的前提不存在

批量执行：交易在下一个区块成交，验证者无法在当前区块内插入交易

对比以太坊MEV：

Uniswap：交易者在mempool里可见 → MEV机器人抢先买入 → 等你交易后再卖出 → 赚取差价

tacit AMM：交易在区块N提交，区块N+1批量成交 → 所有同方向交易同价 → 三明治攻击无法操作

这不是”优化掉”MEV，是从数学结构上消除MEV存在的条件。

6. 原子市场（Atomic Marketplace）

除了AMM，tacit还实现了原子OTC市场——点对点的机密代币交易，用BTC直接支付。

T_AXFER：原子交割

T_AXFER是一种比特币交易类型，在一条交易里同时完成两件事：

机密代币从卖方转到买方（索引器验证Pedersen承诺）

BTC从买方转到卖方（比特币原生脚本验证）

因为两者在同一条比特币交易里，要么同时成功要么同时失败——这就是”原子”的含义。不需要第三方担保，不需要跨交易结算。

T_AXFER_VAR：部分成交

T_AXFER_VAR扩展了T_AXFER，支持可变金额的部分成交。卖家挂出一个”愿意以价格P卖出最多N个代币”的订单，买家可以部分成交。

这个流程也是比特币原生的——不依赖链下订单簿撮合，所有信息在链上。

T_PREAUTH_BID_VAR：离线竞价

更巧妙的机制：买家可以预先授权一个竞价（T_PREAUTH_BID_VAR），即使买家离线，卖家也能与之成交。

应用场景：

买家预先签名”我愿意以价格X购买最多Y个代币”
→ 这个授权发布到链上
→ 任何卖家可以在限额内与之成交
→ 买家不需要在线等待

这类似于限价单，但完全在比特币L1上实现。
