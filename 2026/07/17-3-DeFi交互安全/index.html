DeFi交互安全

识别 Rug Pull、假池与合约陷阱

1. 你交互的不是UI，是合约

很多人以为用DeFi就是在某个网站点按钮。实际上你点的每个按钮，背后都是一笔发往某个智能合约的交易。UI可以随便改、可以伪造，但合约逻辑是确定的。

一个残酷的事实：

你看到的”官网”可能是钓鱼站
你看到的”流动性”可能是假的
你看到的”价格”可能是被操纵的

唯一可靠的是链上合约地址和代码。所有的尽职调查，最终都要落到合约上。

2. Rug Pull的三种套路

Rug Pull（拉地毯）是DeFi最常见的骗局：项目方卷走流动性跑路。有三种常见形式：

套路一：撤走流动性

项目方在DEX上添加流动性（比如100万代币+10 ETH），制造”有交易深度”的假象。等散户买入推高价格后，项目方突然移除全部流动性，代币价格瞬间归零。

套路二：增发砸盘

很多山寨代币有mint权限（增发）。项目方自己持有大量代币，然后突然增发海量代币砸向市场，价格归零。你手里的币还在，但已经一文不值。

套路三：权限后门

更隐蔽的：合约里藏着只有owner能调用的函数——比如blacklist（拉黑）、pause（暂停交易）、setFee（设置100%手续费）。项目方可以随时让代币无法卖出，或把你的币转走。

Rug Pull的共同特征：

流动性未锁定 + 持币高度集中 + owner权限未放弃 = 高危

只要这三项里有一项成立，就需要极度警惕。

3. 假池与假币识别

假池（Fake Pool）是骗子部署一个看起来像Uniswap/Curve的池子，但里面的代币是假币——你买入后根本卖不出去，因为没有人愿意接盘假币。

核对合约地址：在CoinGecko/CoinMarketCap查官方合约地址，对比你交互的池子里的代币地址

看流动性锁定：用Unicrypt/PinkLock查流动性是否锁定、锁到什么时候

看持币分布：前10地址持币占比过高（>50%）要警惕，可能是项目方控盘

看交易历史：新代币、零历史交易、单一大额流动性——都是红旗

4. 合约权限检查

在交互任何代币或协议之前，必须检查合约的权限设置。以下是关键检查项：

Owner权限：owner能否调用mint/blacklist/setFee？能否转移你的币？

流动性锁定：LP Token是否锁定？锁定期到什么时候？

代码验证：合约是否在Etherscan开源验证？未验证的合约一律不碰

代理模式：是否是可升级代理（proxy）？升级权限在谁手里？

一个实用的检查原则：

交互前必查三件事：

  1. 合约是否开源验证（Etherscan上的蓝色勾）
  2. owner是否放弃了mint/blacklist权限
  3. 流动性是否锁定且锁定期足够长

三项有任何一项不满足，就不要交互。

5. 蜜罐合约（Honeypot）

蜜罐（Honeypot）是一种更阴险的陷阱：代币可以正常买入，但卖出函数被篡改——你买进去就永远卖不出来。它专门用来”钓”贪便宜的人。

蜜罐的常见手法：

卖出时扣100%手续费（sell tax设到100%）

黑名单机制：只有特定地址能卖，你的地址被拉黑

转账即燃烧：每次转账销毁99%，根本无法变现

6. 用工具做尽职调查

手动查合约太累，有很多现成工具可以帮你：

GoPlus Security：输入合约地址，自动扫描权限/蜜罐/黑名单风险

Token Sniffer：评分系统，低于一定分数就是 scam

BubbleMaps：可视化持币分布，一眼看出是否集中

Etherscan：看合约是否验证、owner是谁、交易历史

De.Fi / Revoke.cash：检查授权和合约风险

最后一条最重要：先用小额测试。

任何新代币/新协议，先花1美元买，能不能卖出来？能卖再加大。这条习惯能帮你避开90%的蜜罐和假池。
