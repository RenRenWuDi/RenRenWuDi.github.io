事件与日志
Event定义、indexed参数、前端监听与订阅
系列二「Solidity智能合约开发」第4篇
💡 你将学到：Event定义与使用、indexed参数的作用、前端订阅事件、历史事件查询
⏱ 阅读时间：约12分钟

  1. 什么是Event（事件）？
    Event是以太坊日志系统的一部分。合约可以发出事件，事件被记录在交易的日志中。
    Event的作用：

• 记录重要状态变化
• 提供链外获取链上数据的途径
• 比storage读取更省gas
• 前端可以实时监听
Event vs Storage Event: 375-3750 Gas Storage: 2万/写入
数据访问 Event: 仅链外可读 Storage: 链上可读
2. 定义与发出事件
基本语法
Events.sol
// 定义事件
event Transfer(address indexed from, address indexed to, uint256 value);

// 发出事件（需要emit关键字）
function transfer(address to, uint256 amount) public {
// 转账逻辑…
emit Transfer(msg.sender, to, amount);
}

完整示例：代币合约
Token.sol
contract Token {
event Transfer(address indexed from, address indexed to, uint256 value);
event Approval(address indexed owner, address indexed spender, uint256 value);

mapping(address => uint256) public balanceOf;

function transfer(address to, uint256 amount) public {
require(balanceOf[msg.sender] >= amount);
balanceOf[msg.sender] -= amount;
balanceOf[to] += amount;
emit Transfer(msg.sender, to, amount);
}
}

⚠️ 注意：Solidity 0.4.21之前不需要emit关键字，现在必须显式emit。
3. indexed参数：构建索引
每个事件最多有3个indexed参数，这些参数可以用于过滤和查询。
indexed的特点：

• 最多3个（Topics限制）
• 被索引的参数存储在日志的Topics中
• 可以被ethers.js/web3.js过滤查询
• 未indexed的参数存储在数据区，只能获取不能过滤
Topics机制
Topic 0：事件签名的哈希（keccak256）
Topic 1-3：indexed参数的值
Data：未indexed参数的数据
Topics解析.js
event Transfer(address indexed from, address indexed to, uint256 value);

// 这个事件的Topics：
// Topic[0] = keccak256(“Transfer(address,address,uint256)”)
// Topic[1] = from地址
// Topic[2] = to地址
// Data = value（未indexed）

  1. 前端订阅事件
    ethers.js订阅
    subscribe.js
    const { ethers } = require(‘ethers’)

async function listenToEvents() {
const provider = new ethers.JsonRpcProvider(“https://mainnet.infura.io“)

const abi = [“event Transfer(address indexed from, address indexed to, uint256 value)”]
const contract = new ethers.Contract(address, abi, provider)

// 监听所有Transfer事件
contract.on(“Transfer”, (from, to, value) => {
console.log(${from} -> ${to}: ${value})
})
}

过滤特定地址的事件
filtered.js
// 只监听发给某个地址的转账
const filter = contract.filters.Transfer(null, “0x目标地址”)
contract.on(filter, (from, to, value) => {
console.log(收到转账: ${value})
})

// 监听自己的转账
const myFilter = contract.filters.Transfer(null, walletAddress)
contract.on(myFilter, (from, to, value) => {
console.log(我收到了 ${value} 代币！)
})

一次性和持续监听
listeners.js
// once：只监听一次
contract.once(“Transfer”, (from, to, value) => {
console.log(“第一笔转账来了！”)
})

// on：持续监听（记得在不需要时移除）
const handler = (from, to, value) => { console.log(“转账:”, value) }
contract.on(“Transfer”, handler)

// 移除监听器
contract.off(“Transfer”, handler)

  1. 查询历史事件
    可以用queryFilter查询已经发生的事件。
    queryHistory.js
    async function queryHistory() {
    const contract = new ethers.Contract(address, abi, provider)

// 查询所有历史Transfer事件
const events = await contract.queryFilter(“Transfer”)
console.log(共 ${events.length} 笔交易)

// 解析事件数据
events.forEach(event => {
const [from, to, value] = event.args
console.log(${from} -> ${to}: ${value})
})
}

按区块范围和地址过滤
filteredQuery.js
// 查询特定区块范围
const fromBlock = 12345678
const toBlock = 12345700
const events = await contract.queryFilter(“Transfer”, fromBlock, toBlock)

// 只查询发给某个地址的转账
const filter = contract.filters.Transfer(null, “0x目标地址”)
const myEvents = await contract.queryFilter(filter)

  1. Event的实际应用场景
    代币转账记录：ERC20/ERC721的Transfer事件
    NFT铸造与交易：Mint、Trade、Sale事件
    治理投票：ProposalCreated、VoteCast事件
    去中心化交易所：Swap、Deposit、Withdraw事件
    预言机数据：PriceUpdated事件
    Event驱动的前端更新
    TokenWatcher.js
    class TokenWatcher {
    constructor(contract) {
    this.contract = contract
    }

start() {
this.contract.on(“Transfer”, (from, to, value) => {
this.updateBalance(to)
this.showNotification(收到 ${value} 代币)
})
}

stop() {
this.contract.removeAllListeners(“Transfer”)
}
}

  1. 总结

① Event是以太坊日志系统，比storage更省gas
② 使用emit关键字发出事件
③ indexed参数可以构建索引，最多3个
④ 事件签名哈希存储在Topic[0]
⑤ 前端可以用on/once订阅实时事件
⑥ queryFilter可以查询历史事件
