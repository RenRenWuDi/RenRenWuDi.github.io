闪电贷攻击
DeFi最大黑客来源
系列六「智能合约安全攻防」第3篇
💡 你将学到：闪电贷原理、价格操纵攻击、借贷攻击、TWAP防护
⏱ 阅读时间：约13分钟

  1. 闪电贷是什么？
    闪电贷（Flash Loan）是DeFi最强大的工具之一：你可以借走巨量资金，不需要任何抵押品，条件只有一个——必须在同一个交易内归还本金+利息。
    这个限制听起来很安全——如果借了不还，整个交易就回滚，好像什么都没发生过。但问题出在”同一个交易内”：在这短暂的时间里，你可以用借来的钱做任何事。
    闪电贷的核心特性：

• 无需抵押：借多少都行
• 同一交易内必须归还
• 失败则整个交易回滚
• 可借金额理论上没有上限
2. 闪电贷如何运作？
FlashLoanFlow.js
// 闪电贷的三个步骤
// 1. 借款：在交易开始时借走资金
IERC20(USDC).transferFrom(lendingPool, myContract, 1000000e6);
// 2. 操作：用借来的钱做任何事（套利、攻击、swap）
// 3. 还款：在交易结束前归还本金+利息
IERC20(USDC).transfer(lendingPool, 1000000e6 + fee);
// 如果余额不足，整个交易回滚

  1. 闪电贷攻击原理
    闪电贷把”需要很多资本才能做的事”变成了”任何人都能做到”。黑客利用这一点，用借来的巨量资金操纵市场、实施攻击。
    闪电贷攻击的套路：

  2. 借出大量资产

  3. 用这笔钱操纵某个市场的价格

  4. 在被操纵的价格上获利（套利或清算）

  5. 归还借款

  6. 净利润装入口袋

  7. AMM价格操纵攻击
    Uniswap等AMM的价格由流动性池的比例决定。大量买入可以推高价格——这就是闪电贷攻击的基础。
    PriceManipulation.js
    // AMM价格操纵原理
    // Uniswap池子：100 ETH + 200000 USDC
    // 初始价格：200000/100 = 2000 USDC/ETH
    // 攻击：借10000 ETH，用全部ETH换USDC
    // 结果：池子变为 0 ETH + 1200000 USDC
    // 价格：1200000/0.01 = 120美元（暴跌！）
    // 攻击者用低价ETH偿还闪电贷，净赚

  8. 借贷协议清算攻击
    当用户的抵押品价值下跌，低于清算阈值时，任何人都可以清算他们。闪电贷可以用来大批量触发清算。
    借贷清算攻击流程：

  9. 闪电贷借出大量资产

  10. 操纵抵押品价格

  11. 大批用户的抵押率触发清算线

  12. 攻击者以折扣价格获得抵押品

  13. 归还闪电贷，净赚差价

  14. 真实攻击案例
    Cream Finance（2022年1月）：损失1.3亿美元，闪电贷操纵价格
    Aave（2022年3月）：被攻击2500万美元
    Mango Markets（2022年10月）：利用闪电贷操纵MNGO价格获利1.17亿美元
    BNB Chain攻击（2022年10月）：利用跨链漏洞，闪电贷配合
    Aave攻击详解
    攻击者借出大量USDC，用这笔钱推高CRV价格，然后在AAVE上用CRV做抵押品借出更多资产，最后通过Uniswap卖出CRV获利。整个过程在一个交易内完成。

  15. 防御措施
    TWAP（时间加权平均价格）：

使用一段时间的平均价格，而不是即时价格，可以防止价格操纵。Chainlink等预言机提供的价格本身就是TWAP。
流动性限制：

限制单笔操作的流动性，减少价格影响。
延迟清算：

清算不要立即执行，设置延迟窗口。
价格来源多元化：

不要只依赖一个数据源或一个市场。
TWAP防护代码

  1. 总结

① 闪电贷是无抵押、同一交易内归还的贷款
② 攻击核心：借大量资金→操纵价格→在虚假价格上获利→还款
③ AMM价格容易被大额交易操纵
④ 借贷协议清算可以被闪电贷批量触发
⑤ TWAP和多元化数据源是主要防护手段
⑥ 闪电贷攻击已是DeFi损失的最大来源
