稳定币
USDC、DAI与算法稳定币
系列三「DeFi原理与实战」第3篇
💡 你将学到：稳定币类型、USDC/DAI机制、抵押率、算法稳定币风险
⏱ 阅读时间：约12分钟

  1. 什么是稳定币？
    稳定币是一种与美元、欧元等法定货币挂钩的加密货币，其价格保持相对稳定。
    为什么需要稳定币？

• 加密货币波动大，不适合日常支付
• 法币难以直接与DeFi交互
• 交易所计价需要稳定资产
• 避险：牛市中锁定利润
2. 稳定币的三种类型
法币抵押型（USDC、USDT）—— 1:1美元储备
加密资产抵押型（DAI）—— 超额抵押ETH等资产
算法稳定币（UST、Frax）—— 无抵押，靠算法维持
3. USDC：法币抵押型
USDC由Circle发行，每发行1枚USDC，就有1美元储备。
USDC机制：

• 中心化发行：Circle公司
• 1:1美元储备，定期审计
• 铸造/销毁需要KYC
• 透明度高，但依赖中心化主体
合约示例
USDC.sol
contract USDC {
string public name = “USD Coin”;
string public symbol = “USDC”;
uint8 public decimals = 6;
uint256 public totalSupply;
mapping(address => uint256) public balanceOf;
mapping(address => mapping(address => uint256)) public allowance;
mapping(address => bool) public minters; // 有权限的铸造者
function mint(address to, uint256 amount) external {
require(minters[msg.sender], “Not minter”);
totalSupply += amount;
balanceOf[to] += amount;
}
function burn(uint256 amount) external {
require(balanceOf[msg.sender] >= amount);
totalSupply -= amount;
balanceOf[msg.sender] -= amount;
}
}

  1. DAI：加密资产抵押型
    DAI由MakerDAO发行，通过超额抵押ETH等加密资产生成。
    DAI机制：

• 去中心化：由MakerDAO治理
• 超额抵押：抵押率通常>150%
• 抵押ETH借DAI，利息浮动
• 稳定性费用调节供需
CDP（抵押债仓）流程
步骤1：存入ETH作为抵押品
步骤2：生成DAI（债务）
步骤3：归还DAI+利息，取回抵押品
步骤4：抵押率低于清算线，抵押品被清算
5. 算法稳定币
算法稳定币尝试用算法替代抵押品来维持价格稳定。
算法稳定币的问题（UST崩盘教训）：

• LUNA/UST在2022年5月崩盘
• UST脱锚，一度跌至0.01美元以下
• LUNA从80美元跌至0.0001美元
• 几百亿美元市值蒸发
Terra UST机制（已崩盘）
UST.js
// UST的铸币机制
// 当1 UST 1美元时：铸造UST卖出盈利
mintUST(amount); // 铸造UST
sellUST(amount); // 卖出UST获得USD
// 当1 UST 1美元时：销毁UST获得LUNA
burnUST(amount); // 销毁UST
redeemLUNA(value); // 获得等值LUNA
// 问题：LUNA无上限供应，抛压导致价格归零

Frax：部分算法稳定币
Frax是混合型稳定币，部分由抵押品支持，部分由算法调节。
Frax的创新：

• 抵押率不是100%，而是动态调整
• 当USDC充足时，抵押率高
• 当市场紧张时，算法比例增加
• 被称为半算法稳定币
6. 对比总结
USDC：中心化、简单、依赖信任
DAI：去中心化、安全、超额抵押
算法稳定币：高风险高收益、可能归零
7. 总结

① 稳定币是与法币挂钩的加密货币，用于支付和DeFi
② USDC由美元储备支持，中心化但可靠
③ DAI由加密资产超额抵押，去中心化
④ 算法稳定币尝试用算法替代抵押品
⑤ UST的崩盘提醒我们算法稳定币的高风险
⑥ 选择稳定币要考虑去中心化程度和风险
