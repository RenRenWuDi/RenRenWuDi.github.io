智能合约安全工具
Slither、Certora、Mythril实战
系列六「智能合约安全攻防」第5篇（完结篇）
💡 你将学到：Slither静态分析、Mythril符号执行、Certora形式化验证、安全审计流程
⏱ 阅读时间：约13分钟

  1. 为什么需要安全工具？
    智能合约一旦部署就无法修改，任何bug都可能导致资金损失。传统代码可以打补丁升级，但合约不行——上线前的安全审计至关重要。
    自动化安全工具可以在几秒钟内扫描整个合约代码库，发现常见漏洞。它们不能替代专业审计，但可以大幅减少低级错误。
    安全审计的层次：

  2. 自动化工具扫描（CI/CD集成）

  3. 人工代码审计

  4. 第三方专业审计公司

  5. Bug Bounty（漏洞赏金）

自动化工具是第一道防线，也是成本最低的。
2. Slither（Trail of Bits）
Slither是以太坊生态最流行的静态分析工具，由Trail of Bits开发。它用Python编写，基于Solidity编译器的中间表示（IR），分析精确度高。
安装与运行
Terminal命令
pip install slither-analyzer
slither . # 分析当前目录
slither ./contracts/MyContract.sol –filter-path ./contracts
slither . –detect reentrancy-eth,unchecked-lowlevel

常见检测器
reentrancy-eth：重入攻击
unchecked-lowlevel：未检查的外部调用
unused-return：未检查返回值
tx-origin：tx.origin使用
shadowing-state：状态变量遮蔽
生成详细报告
Terminal命令
slither . –json-report report.json
slither . –markdown-report report.md
slither . –sarif-report report.sarif

printer功能
Slither的Printer可以输出合约的结构信息。
Terminal命令
slither . –print contract-summary
slither . –print function-summary
slither . –print call-graph
slither . –print inheritance
slither . –print variable-order

  1. Mythril
    Mythril是用符号执行（Symbolic Execution）技术的安全分析工具，由ConsenSys Diligence开发。相比静态分析，Mythril可以探索更多执行路径。
    Terminal命令
    pip install mythril
    myth analyze ./contracts/MyContract.sol
    myth analyze ./contracts/MyContract.sol –solc-json solc.json
    myth detect ./contracts/MyContract.sol -o text

Mythril vs Slither对比
Slither：静态分析，快速，CI/CD友好
Mythril：符号执行，深层漏洞，准确率更高
两者配合使用效果最好
4. Certora（形式化验证）
Certora是形式化验证工具，可以数学上证明代码的正确性。比传统测试更强大——测试告诉你漏洞存在，形式化验证告诉你漏洞不存在。
Certora的CVL（Certora Verification Language）让你编写规格（specifications），工具会自动验证代码是否满足规格。
spec.cvl
// Certora规范示例（CVL）
rule noNegativeBalance(address user) {
env e;
uint256 balanceBefore = balanceOf(e, user);
callWithdraw(e, user, balanceBefore);
uint256 balanceAfter = balanceOf(e, user);
assert balanceAfter >= 0, “Balance cannot be negative”;
}

Terminal命令

Certora命令

certoraRun MyContract.sol –rule noNegativeBalance
certoraRun MyContract.sol –spec spec.cvl

生成验证报告

certoraRun MyContract.sol –verify MyContract.spec –msg “Verifying balance rules”

形式化验证的价值
传统测试：发现bug → 修复 → 重新测试，永远无法证明没有bug。
形式化验证：编写规格 → 自动证明 → 数学上保证代码符合规格。
形式化验证适合的场景：

• 关键函数：withdraw、transfer、mint
• 金融逻辑：利率计算、清算阈值
• 权限控制：角色验证、访问控制
5. Echidna（模糊测试）
Echidna是智能合约的模糊测试工具，自动生成随机交易序列来测试合约，找出可能导致异常的行为。
Terminal命令
npm install -g echidna
echidna . –contract MyContract
echidna . –config echidna.yaml
echidna . –coverage # 统计测试覆盖率

echidna.yaml
// echidna.yaml 配置文件
testMode: assertion
timeout: 3600
corpusDir: corpus
sequenceLength: 100
testLimit: 50000

  1. 完整的DevSecOps流程
    将安全工具集成到CI/CD流程中，每次提交代码都自动扫描。
    DevSecOps安全流程：

• GitHub Actions / GitLab CI
• 每次PR：Slither静态分析
• 每次合并：Mythril深度扫描
• 关键版本：Certora形式化验证
• 上线前：第三方审计
GitHub Actions集成示例
security.yml
name: Security Scan
on: [push, pull_request]
jobs:
slither:
runs-on: ubuntu-latest
steps:

  • uses: actions/checkout@v3
  • uses: python/setup-python@v4
    with: python-version: 3.10
  • run: pip install slither-analyzer
  • run: slither . –fail-on: medium
  1. 安全工具局限性
    自动化工具无法发现业务逻辑漏洞
    无法检测外部依赖的漏洞
    无法保证与其他合约的交互安全
    形式化验证需要编写精确的规格
    工具报告的误报率需要人工过滤
  2. 总结

① 智能合约安全工具是上线前第一道防线
② Slither是静态分析首选，CI/CD集成友好
③ Mythril用符号执行发现深层漏洞
④ Certora提供数学级别的正确性保证
⑤ Echidna模糊测试覆盖随机边界情况
⑥ DevSecOps流程：每次提交都自动扫描
