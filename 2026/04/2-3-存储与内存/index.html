存储与内存
storage、memory、calldata的区别与使用场景
系列二「Solidity智能合约开发」第3篇
💡 你将学到：三种数据位置的区别、Gas成本、常见错误与最佳实践
⏱ 阅读时间：约12分钟

  1. 三种数据位置概述
    Solidity中，复杂类型（数组、结构体、映射）需要指定数据位置。
    三种数据位置：

• storage：永久存储在区块链上，最昂贵
• memory：临时存储在内存中，函数执行完释放
• calldata：只读，存储函数参数，最省gas
对比表
生命周期 storage: 永久 memory: 临时 calldata: 临时
Gas成本 storage: 最高（2万/字） memory: 中等 calldata: 最低
可修改性 storage: 可读写 memory: 可读写 calldata: 只读
使用场景 storage: 状态变量 memory: 临时计算 calldata: 函数参数
2. storage：永久存储
storage用于存储合约的状态变量，数据永久保存在区块链上。
storage特点：

• 读写操作消耗大量Gas
• 写操作（SSTORE）约2万Gas
• 修改已存在的存储槽约5千Gas
• 读操作（SLOAD）约100Gas（冷读）或100Gas（热读）
状态变量默认storage
DataLocation.sol
contract StorageExample {
// 状态变量默认在storage
uint256 public count; // storage
mapping(address => uint) balances; // storage
string public name; // storage

function increment() public {
count++; // 修改storage，消耗gas
}
}

显式使用storage
DataLocation.sol
contract ExplicitStorage {
struct User {
string name;
uint256 balance;
}

mapping(address => User) public users;

function updateUser(address userAddr, string memory newName) public {
// 显式声明storage引用
User storage user = users[userAddr];

// 通过引用修改，直接修改storage
user.name = newName;
// 等价于: users[userAddr].name = newName;
}
}

💡 技巧：使用storage引用可以避免重复的映射查找，节省gas。
3. memory：临时内存
memory用于存储临时数据，函数执行完毕后自动释放。
memory特点：

• 按字（32字节）计费，前724字免费
• 扩展memory时按平方计费（防止滥用）
• 适合临时计算和中间结果
函数参数和局部变量
DataLocation.sol
contract MemoryExample {
function processArray(uint[] memory arr) public pure returns (uint) {
// arr在memory中
uint sum = 0;

for (uint i = 0; i < arr.length; i++) {
sum += arr[i];
}

return sum;
}

function createMemoryArray() public pure returns (uint[] memory) {
// 在memory中创建新数组
uint[] memory newArray = new uint;
newArray[0] = 1;
newArray[1] = 2;
return newArray;
}
}

memory vs storage 拷贝
DataLocation.sol
contract CopyExample {
uint[] public stateArray; // storage

function badCopy() public view {
// 错误：将整个数组从storage拷贝到memory
uint[] memory memArray = stateArray; // 昂贵的拷贝！
// …
}

function goodApproach() public view returns (uint) {
// 正确：只读取需要的元素
return stateArray[0]; // 只读一个元素
}
}

⚠️ 警告：从storage拷贝数据到memory非常昂贵！尽量避免大数组的完整拷贝。
4. calldata：只读参数区
calldata是特殊的只读数据位置，仅用于外部函数的参数。
calldata特点：

• 只读，无法修改
• 不占用memory空间
• 比memory更省gas
• 仅用于external函数的参数
calldata vs memory
DataLocation.sol
contract CalldataExample {
// 使用calldata - 更省gas
function sumWithCalldata(uint[] calldata arr) external pure returns (uint) {
uint sum = 0;
for (uint i = 0; i < arr.length; i++) {
sum += arr[i]; // 只读访问
}
return sum;
}

// 使用memory - 会复制数据
function sumWithMemory(uint[] memory arr) public pure returns (uint) {
uint sum = 0;
for (uint i = 0; i < arr.length; i++) {
sum += arr[i];
}
return sum;
}
}

何时使用calldata？
外部函数（external）的参数
不需要修改的数组/结构体参数
希望节省gas的场景
5. 常见错误与最佳实践
错误1：默认storage导致意外修改
DataLocation.sol
contract BugExample {
struct User { string name; uint balance; }
mapping(address => User) public users;

function bug() public {
// 错误：默认是storage，指向已存在的槽
User storage user;
user.name = “Alice”; // 修改了随机的storage位置！
}

function correct() public {
// 正确：显式使用memory
User memory user;
user.name = “Alice”; // 只在memory中
users[msg.sender] = user; // 显式保存
}
}

错误2：memory数组无法动态扩展
DataLocation.sol
contract ArrayBug {
function bug() public pure {
uint[] memory arr = new uint;
arr[0] = 1;
// arr.push(4); // 错误！memory数组没有push方法
}
}

最佳实践
函数参数尽量使用calldata（external函数）
避免从storage到memory的大数组拷贝
局部变量明确指定memory或storage
多次访问同一storage位置时，使用storage引用
不需要持久化的数据使用memory
6. 完整示例：优化的代币转账
DataLocation.sol
contract OptimizedToken {
mapping(address => uint256) private balances;

// 使用calldata节省gas
function batchTransfer(
address[] calldata recipients,
uint256[] calldata amounts
) external {
require(recipients.length == amounts.length, “Length mismatch”);

// 使用storage引用避免重复查找
uint256 senderBalance = balances[msg.sender];
uint256 total = 0;

for (uint i = 0; i < amounts.length; i++) {
total += amounts[i];
}

require(senderBalance >= total, “Insufficient balance”);

// 只在最后修改一次storage
balances[msg.sender] = senderBalance - total;

for (uint i = 0; i < recipients.length; i++) {
balances[recipients[i]] += amounts[i];
}
}
}

  1. 总结

① storage：永久存储，最昂贵，用于状态变量
② memory：临时存储，中等成本，用于临时计算
③ calldata：只读参数区，最省gas，用于external函数
④ 避免大数组从storage到memory的拷贝
⑤ 多次访问storage时使用引用优化
⑥ 合理选择数据位置可以显著降低gas成本
