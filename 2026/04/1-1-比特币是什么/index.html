比特币是什么？——从贝壳到数字黄金
一文搞懂比特币的设计思想、UTXO模型与转账原理
💡 你将学到：比特币解决了什么问题、UTXO模型、交易验证逻辑
⏱ 阅读时间：约10分钟

  1. 先问一个问题：钱是什么？
    远古时候，人们用贝壳交易。后来用黄金，因为稀缺。
    但黄金携带不方便，于是有了银行——你把黄金存进去，银行给你一张欠条，这张欠条可以流通。
    银行的问题：中心化、可能被冻结、可能超发。

比特币想做的事：用代码取代银行
2. 比特币的核心理念
中本聪在2008年提出比特币，白皮书标题很直接：
Bitcoin: A Peer-to-Peer Electronic Cash System
比特币：一种点对点的电子现金系统
三个关键词：
去中心化 —— 没有银行，没有央行
点对点 —— 我转给你，不需要中间人
电子现金 —— 数字形式，但”现金”的特性：匿名、不可逆
3. UTXO模型：比特币的记账方式
以太坊用的是账户模型（像银行账户），而比特币用的是 UTXO。
什么是UTXO？
UTXO = Unspent Transaction Output（未花费交易输出）
通俗理解：每一笔比特币交易，都是在销毁一些”币”并创造一些新的”币”。
就像你用一张100元钞票买咖啡，收银员找给你70元——那张100元被”销毁”了，变成30元和70元两张”新钞票”。
代码示例：UTXO交易模拟
utxo-demo.js
// UTXO：未花费的交易输出
class UTXO {
constructor(txHash, index, amount, owner) {
this.txHash = txHash // 来源交易哈希
this.index = index // 输出位置
this.amount = amount // 金额（1 BTC = 1亿聪）
this.owner = owner // 所有者地址
this.spent = false
}
}

// 交易：输入UTXO，输出新的UTXO
class Transaction {
constructor(inputs, outputs) {
this.inputs = inputs
this.outputs = outputs
}
}

// 场景：张三给李四转 0.5 BTC
const utxo1 = new UTXO(‘tx001’, 0, 100000000, ‘张三’) // 1 BTC

const tx = new Transaction(
[utxo1],
[
{ address: ‘李四’, amount: 50000000 }, // 李四得 0.5 BTC
{ address: ‘张三’, amount: 49999000 } // 张三得找零
]
)

utxo1.spent = true
console.log(‘交易完成！’)

手续费：100 satoshi

  1. 比特币地址是怎么来的？
    比特币地址不是随机生成的，而是通过密码学推导：
    🔢 随机数
    🔑 私钥 ↓
    🔐 公钥 ↓
    📍 地址
    过程分解
    生成随机私钥（256位随机数）
    用私钥生成公钥（椭圆曲线加密 secp256k1）
    公钥哈希（SHA256 + RIPEMD160）
    Base58Check编码得到地址
    代码示例：生成测试地址
    utxo-demo.js
    const crypto = require(‘crypto’)

// 生成随机私钥（32字节）
const privateKey = crypto.randomBytes(32)
console.log(‘私钥:’, privateKey.toString(‘hex’))

// 简化版地址生成（生产环境请用 bitcoinjs-lib）
function simpleAddress(key) {
return ‘1’ + crypto
.createHash(‘sha256’)
.update(key)
.digest(‘hex’)
.substring(0, 24)
}

const address = simpleAddress(privateKey.toString(‘hex’))
console.log(‘地址:’, address)

⚠️ 注意：真实比特币地址需要椭圆曲线运算。推荐库：bitcoinjs-lib 或 ethers.js
5. 比特币转账的完整流程
📤 用户发起转账 ↓
📝 钱包构建交易 ↓
🔐 用私钥签名 ↓
📡 广播到网络 ↓
⛏ 矿工验证打包 ↓
✅ 获得确认
一个关键点
比特币没有”账户余额”这个概念。

查询余额 = 遍历整个链，找出所有属于你的未花费UTXO，加起来就是你的余额。

所以比特币钱包同步很慢——它要扫描整条链。
6. 比特币 vs 银行账户
余额记录方式 比特币：UTXO集合 银行：账户余额
转账确认时间 比特币：约1小时（6个区块） 银行：实时到账
身份凭证 比特币：私钥 银行：身份证+密码
资金安全 比特币：私钥在手=绝对掌控 银行：可能被冻结
可编程性 比特币：脚本能力有限 银行：API丰富
7. 总结

① 比特币解决了去中心化价值转移的问题
② UTXO是比特币记账的核心，理解它是入门关键
③ 私钥 = 比特币所有权，丢了就真的没了
④ 比特币是”数字黄金”，以太坊是”数字石油”
